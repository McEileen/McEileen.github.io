Looping in Ruby vs. Looping in JavaScript

Differences and Similarities

August 20, 2015

As I find it easier to learn from coding exercises than from a textbook, I decided to re-do the Codecademy Ruby track and JavaScript track portions on looping. The conceptual bases of while and for loops are similar in both languages. However, there are differences in syntax that are important to remember.

In both Ruby and JavaScript, while loops are ideal for when you want to repeat an action but don't know how many times you'll have to repeat that action. Also in both languages, for loops are ideal for when you know how many times you need to loop.

Here is an example of a Ruby while loop.

counter = 1 while counter < 11 puts counter counter = counter +1 end

Here is an example of a JavaScript while loop.

count = 0; var loop = function(){ while(count < 3) { console.log("I'm looping!"); count ++; } };

Here is an example of a Ruby for loop.

for num in 1...10 puts num end

Here is an example of a JavaScript for loop.

for (var counter = 1; counter < 11; counter++){ console.log(counter); }

There are certain looping techniques available in one language but not in the other. For example, Ruby has the until loop, which is a complement to the while loop. You can think of it as a backwards while loop. See an example below.

i = 0 until i == 6 i += 1 end puts i

In JavaScript, when you want to ensure that your loop runs at least one time no matter what, you can run a 'do'/'while' loop.

var loopCondition = false; do { console.log("I'm gonna stop looping 'cause my condition is" + loopCondition + "!"); } while (loopCondition);
