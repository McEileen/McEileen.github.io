Today at post-lunch prep (PLP) we did a problem in bash, which was a PLP first for me. The problem, 193. Valid Phone Numbers, asked us to write a single-line bash expression to find valid phone numbers. My pair and I started by using a few helpful websites, namely, www.regexr.com, www.rubular.com, and ye olde www.stackoverflow.com, to devise a regular expression that would match valid phone numbers. See below for our solution. 

      
        while read -r line; do if [[ $line =~ ^(\([0-9]{3}\) |[0-9]{3}-)[0-9]{3}-[0-9]{4}$ ]]; then echo $line; fi; done < file.txt;

Disclaimer: Another Recurser came up with a faster solution by using grep instead of a while loop. According to their explanation, grep is more efficient than while loops for searching multiple lines in a file. Good to know!
